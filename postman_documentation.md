
# Documentación de API para Campañas (Postman)

Esta documentación describe cómo probar la API de creación de campañas utilizando Postman.

## 1. Autenticación (Obtener Token Sanctum)

Antes de crear una campaña, necesitas autenticarte para obtener un token de acceso.

**Endpoint:** `POST http://127.0.0.1:8000/api/login`
**Headers:**
- `Content-Type`: `application/json`

**Body (raw, JSON):**
```json
{
    "email": "test@example.com",
    "password": "password"
}
```
**Notas:**
- Asegúrate de tener un usuario registrado en tu base de datos (puedes usar el seeder `TestUserSeeder.php` si lo tienes).
- Guarda el `plainTextToken` de la respuesta para usarlo en las solicitudes posteriores.

## 2. Crear Campaña (con subida de archivos)

Una vez autenticado, puedes crear una nueva campaña, incluyendo la subida de archivos de segmentación.

**Endpoint:** `POST http://127.0.0.1:8000/api/campaigns`
**Headers:**
- `Accept`: `application/json`
- `Authorization`: `Bearer {{your_sanctum_token}}` (Reemplaza `{{your_sanctum_token}}` con el token obtenido en el paso de autenticación)

**Body (form-data):**
Selecciona `form-data` en la pestaña "Body" de Postman.
